1. Move the navigation buttons outside the tabs
In each phase template (e.g. phase1.html, phase2.html‚Ä¶), you likely have something like:

  <button class="tab-button active" data-tab="thinking">Agent Thinking</button>
  <button class="tab-button" data-tab="outputs">Outputs</button>
</div>
<div class="tab-panel active" id="tab-thinking">
  <!-- thinking content -->
  <!-- Run Phase + Proceed to Phase 2 buttons currently here -->
</div>
<div class="tab-panel" id="tab-outputs">
  <!-- outputs content -->
</div>
And your ‚ÄúProceed to Phase 2‚Äù button is probably inside tab-thinking.
Change it to this structure:

  <button class="tab-button active" data-tab="thinking">Agent Thinking</button>
  <button class="tab-button" data-tab="outputs">Outputs</button>
</div>
<div class="tab-panel active" id="tab-thinking">
  <!-- thinking content ONLY (status + steps + run button) -->

Key ideas:
The footer is outside both tab panels, so it shows in both Agent Thinking and Outputs views.
current_phase_completed should be passed from the view function (True if phaseX_completed for that case is true).
The next-phase button is disabled until the phase has run, but still visible on both tabs.
You should repeat this pattern (or include a shared partial) for all four phase templates.
2. View function changes (minimal)
In each GET /cases/{case_id}/phases/{phase_no} view, set current_phase_completed:
@app.get("/cases/{case_id}/phases/{phase_no}")
async def view_phase(request: Request, case_id: int, phase_no: int):
    case = db_session.get(Case, case_id)
    # ...
    if phase_no == 1:
        current_phase_completed = case.phase1_completed
    elif phase_no == 2:
        current_phase_completed = case.phase2_completed
    elif phase_no == 3:
        current_phase_completed = case.phase3_completed
    else:
        current_phase_completed = case.phase4_completed
    return templates.TemplateResponse(
        "phase1.html" if phase_no == 1 else ...,
        {
            "request": request,
            "case": case,
            "current_phase": phase_no,
            "current_phase_completed": current_phase_completed,
            # plus your other context (outputs, thinking, etc.)
        }
    )
This is what the template uses to decide whether to enable/disable the ‚ÄúProceed‚Äù button.

4. Sanity check your assumption
You were assuming: ‚ÄúButton exists; why isn‚Äôt it visible on the Outputs tab?‚Äù
Reality: it‚Äôs scoped to the Thinking panel only. Tabs hide the entire panel, not just the text.
By lifting navigation into a shared footer, you:
Avoid duplicating logic across tabs,
Make sure the wizard controls are always in the same place,
And make the interface feel more deliberate (which is exactly the ‚Äúintuitive‚Äù feedback your team wants).
